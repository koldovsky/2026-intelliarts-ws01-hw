# Architecture

## High-level Architecture

Excalidraw is a **Yarn-workspace monorepo** that separates concerns into discrete packages consumed by a standalone React application.

```
excalidraw-monorepo/
├── excalidraw-app/          # Production app (Vite + React + PWA)
└── packages/
    ├── excalidraw/          # Core library exported to npm (@excalidraw/excalidraw)
    ├── element/             # Element types, mutation helpers, rendering
    ├── common/              # Shared constants, utilities, math helpers
    ├── math/                # Geometric primitives (points, vectors, curves)
    ├── utils/               # Generic helpers (not geometry-specific)
    └── fractional-indexing/ # Ordered-index helpers for collaborative ordering
```

`excalidraw-app` is the deployed product; `packages/excalidraw` is the publishable library. Consumer code (e.g. `excalidraw-app`) mounts `<Excalidraw>` from `@excalidraw/excalidraw` and optionally provides callbacks, initial data, and UI extension slots.

## Data Flow

```
User input (pointer/keyboard)
        │
        ▼
  App.tsx event handlers
  (onPointerDown, onKeyDown, …)
        │
        ├──► ActionManager.executeAction(action, …)
        │         │
        │         ▼
        │    action handler returns { appState?, elements?, … }
        │         │
        ▼         ▼
  App.setState / Scene.mutateElement
        │
        ▼
  React re-render triggers Renderer
        │
        ├──► staticScene  →  offscreen CanvasRenderingContext2D (static layer)
        └──► interactiveScene → overlay canvas (selection handles, cursors)
```

Mutations never touch element objects directly — they go through `mutateElement()` in `packages/element/src/mutateElement.ts`, which increments `version` + `versionNonce` and schedules a store capture.

## State Management

### AppState (`packages/excalidraw/appState.ts`)

A single large plain object managed by `App` component state (`this.state`). Contains:

- Active tool (`activeTool`)
- Current item style properties (`currentItemStrokeColor`, `currentItemFillStyle`, …)
- UI toggles (grid, snap, theme, sidebar)
- Collaboration metadata (`collaborators`)
- Zoom, scroll offsets, canvas dimensions

`AppState` is **not** persisted directly — only the subset needed for undo/redo and file save is stored.

### Scene / Element Store (`packages/element/src/store.ts` + `Scene.ts`)

`Scene` holds the canonical `Map<id, OrderedExcalidrawElement>` (a `SceneElementsMap`). The `Store` tracks versioned snapshots (`StoreSnapshot`) and emits `StoreChange` events that feed the history system.

### History (`packages/excalidraw/history.ts`)

`HistoryDelta` extends `StoreDelta` to record the diff between two snapshots. Undo/redo replay deltas against the current snapshot via `HistoryDelta.applyTo(elements, appState, snapshot)`.

### Jotai atoms (`packages/excalidraw/editor-jotai.ts`)

Lightweight reactive state for UI concerns (sidebar, dialog visibility) that don't belong in the monolithic `AppState`. Wrapped by `EditorJotaiProvider`.

## Rendering Pipeline

Excalidraw renders to **two stacked `<canvas>` elements** to avoid redrawing static content on every pointer move.

### Static canvas (`staticScene.ts`)

Renders all non-interactive scene content on each committed state change:

1. `bootstrapCanvas()` — clears + applies scroll/zoom transform
2. Grid lines (optional)
3. For each non-deleted element: `renderElement(element, …)` via `@excalidraw/element`
   - Shapes are generated by **roughjs** and cached in `ShapeCache` (keyed by element `seed`)
   - Text elements use the browser Canvas2D text API
   - Images are drawn from the `files` blob map
4. Frame clip regions applied where needed

### Interactive canvas (`interactiveScene.ts`)

Renders ephemeral overlay on every pointer event (no caching):

- Selection boxes and multi-selection rectangle
- Transform handles
- Binding indicators and snap lines
- Collaborator cursors and selections
- New element being drawn (`renderNewElementScene.ts`)

### SVG export (`staticSvgScene.ts`)

A parallel path that serialises the same element tree into an `<svg>` DOM node instead of canvas commands, used by the export dialog.

### RoughJS integration

Shape outlines (strokes, fills) are generated by `rough.generator()` in `renderElement.ts`. The result is cached in `ShapeCache` keyed by `element.seed` so identical elements don't regenerate geometry on each frame.

## Package Dependencies

```
excalidraw-app
    └── @excalidraw/excalidraw
            ├── @excalidraw/element
            │       ├── @excalidraw/common
            │       └── @excalidraw/math
            ├── @excalidraw/common
            ├── @excalidraw/math
            └── @excalidraw/fractional-indexing

@excalidraw/common      — zero internal deps (leaf)
@excalidraw/math        — zero internal deps (leaf)
@excalidraw/fractional-indexing — zero internal deps (leaf)
@excalidraw/utils       — zero internal deps (leaf)
@excalidraw/element     — depends on common + math
@excalidraw/excalidraw  — depends on element + common + math + fractional-indexing
```

Build order enforced by root `package.json` scripts: `common` → `fractional-indexing` → `math` → `element` → `excalidraw`.

Key external runtime dependencies:

| Dependency | Purpose |
|---|---|
| `roughjs` | Hand-drawn stroke/fill generation |
| `react` / `react-dom` | Component tree and rendering |
| `jotai` | Lightweight atom-based UI state |
| `nanoid` | Element ID generation |
| `lodash.throttle` | Pointer event throttling |
| `browser-fs-access` | File System Access API (save/open) |
